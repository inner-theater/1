## 18:30 平行时空来信改为盲测设计（方案A）
- **问题**：三封信各自独立随机选 A 或 B，可能全是一个选择，没有盲测意义
- **改为方案A（盲测）**：
  - 只在 `ai.js` 中随机一次，确定 `chosen`（A或B），三封信都写 `chosen` 的未来
  - 用户读信时**不知道**这是哪条路的未来，纯粹凭文字感受
  - 解读时揭晓：「你刚才读的三封信来自XX的未来」
  - Edge Function 支持从 context 接收 `chosen`
